Modelleringsprinciper i förslaget

  • Alla översättningsbara texter sparas i Translation via EntityType + EntityId + FieldName + Culture.
  • Taggar hanteras generiskt med TagGroup, Tag och TagAssignment för att kunna återanvändas mellan profiler, artiklar, loggbok och träffar.
  • NameCode/TextCode används i primärentiteter för nyckelbaserade texter som översätts via Translation.
  • Checklistor kan baseras på en ChecklistTemplate för återanvändbara mallar med fördefinierade punkter.
  • User-auth följer ASP.NET Core Identity (Blazor Individual Auth) med rollerna User och Admin.
  • Profile har separat relation för Vehicle, Child samt InvitedByProfileId.
  • Articles redigeras/lagras i Storyblok CMS och länkas via CmsProvider + CmsEntryId.

Pragmatisk separation: domän vs databas

Ett mellanläge för implementation: behåll SQL-modellen som den är, men exponera enklare domänobjekt i applikationen och lägg mappning i repository-lagret. Då minskar kopplingen till EntityType/FieldName utan att öka komplexiteten för mycket.

Domän (appkod) Databas (SQL) Kommentar
Checklist.Name Checklist.NameCode + Translation Repository hämtar kultur och returnerar färdig text.
Meetup.Description Meetup.DescriptionCode + Translation Domän slipper känna till kod-nycklar.
Article.Tags[] Tag + TagAssignment Databas kan vara generisk även om domänen är tydlig.
