Modelleringsprinciper i förslaget

  • Alla översättningsbara texter sparas i Translation via EntityType + EntityId + FieldName + Culture.
  • Fält märkta med _t-suffix i diagrammet (t.ex. Name_t, Description_t) är virtuella – de visas på entiteten i diagrammet för läsbarhetens skull, men lagras inte i primärtabellen. Värdet hämtas alltid från Translation-tabellen.
  • Taggar hanteras generiskt med TagGroup, Tag och TagAssignment för att kunna återanvändas mellan profiler, artiklar, loggbok och träffar.
  • Primärentiteter lagrar inga översättningsbara textfält; dessa finns enbart i Translation.
  • Checklistor kan baseras på en ChecklistTemplate för återanvändbara mallar med fördefinierade punkter.
  • User-auth följer ASP.NET Core Identity-tabellerna AspNetUsers, AspNetRoles och AspNetUserRoles (Blazor Individual Auth) med roller som User och Admin.
  • Profile har separata relationer för Vehicle, Relative (t.ex. Child, Husband, Wife), Pet samt InvitedByProfileId.
  • Articles redigeras/lagras i Storyblok CMS och länkas via CmsProvider + CmsEntryId.

Pragmatisk separation: domän vs databas

Implementation: gör SQL-modellen konsekvent genom att låta språkfält ligga i Translation och returnera färdiga domänobjekt från repository-lagret. Domänen använder bara vanliga fält som Name/Description, medan repository hanterar EntityType/FieldName/Culture internt.

ℹ️

Om _t-suffix i diagrammet ovan

Fält som slutar på _t (t.ex. Name_t, Title_t) är inte kolumner i primärtabellen. De visas på respektive entitet i diagrammet enbart för att göra det tydligt vilka fält som är översättningsbara – men de lagras i sin helhet i Translation-tabellen och hämtas via EntityType + EntityId + FieldName + Culture.

Domän (appkod) Databas (SQL) Kommentar
Checklist.Name Translation(EntityType=Checklist, FieldName=Name) Primärtabellen har inte fältet; värdet kommer alltid från Translation.
Meetup.Description Translation(EntityType=Meetup, FieldName=Description) Inget språkfält finns i primärtabellen; allt språk styrs via Translation-kontrakt.
Article.Tags[] Tag + TagAssignment Databas kan vara generisk även om domänen är tydlig.

Translation-kontrakt (EntityType / FieldName)

Dokumentera och validera tillåtna kombinationer i applikationen. Detta är källan till sanning för vilka fält som får översättas.

EntityType Tillåtna FieldName Kommentar
Profile DisplayName, Bio, HomeRegion Visningsfält i profil.
Checklist Name, Description Inga Name/Description i primärtabellen (endast i Translation).
ChecklistItem Text En text per rad och kultur.
ChecklistTemplate Name, Description, Category Återanvändbar mall med översatta etiketter.
Meetup Name, Description Plats/tider ligger kvar i primärtabell.
MeetupAgendaItem Title, Description Rubrik + beskrivning per programpunkt.
Article Title, Excerpt, Body Om ni inte hämtar dessa direkt från CMS.
TagGroup, Tag Name Delad taxonomi med översatta etiketter.

Rekommenderade constraints/index:

  • Unikt index på (EntityType, EntityId, FieldName, Culture).
  • CHECK/enum-lista för tillåtna EntityType.
  • Validering av tillåtna FieldName per EntityType i app/repository.
  • Fallback-regel vid läsning: sv-SE -> sv -> defaultCulture.
